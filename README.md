# md-web

Lightweight Go web server that serves Markdown files as styled HTML pages. No build step, no JavaScript — just point it at a directory of `.md` files and go.

## Features

- Markdown to HTML rendering via [Goldmark](https://github.com/yuin/goldmark)
- GitHub Flavored Markdown (tables, strikethrough, autolinks, task lists)
- Syntax highlighting for code blocks
- Wiki links (`[[Page]]`)
- Math rendering (MathML)
- Mermaid diagrams
- YAML frontmatter for metadata
- Base URL rewriting for subpath deployments
- Optional in-memory page cache
- Mobile-friendly, no-JS HTML output
- Single binary, no external dependencies at runtime

## Install

```bash
go install github.com/reddec/md-web@latest
```

Docker:

```bash
docker run -v ./docs:/data -p 8080:8080 ghcr.io/reddec/md-web
```

## Usage

```bash
# Serve current directory on :8080
md-web

# Serve a specific directory
md-web -d /path/to/markdown

# Enable caching and title display
md-web -c -t

# Serve under a subpath
md-web -B /docs
```

## Configuration

| Flag | Env | Default | Description |
|------|-----|---------|-------------|
| `-b, --bind` | `BIND` | `:8080` | HTTP server bind address |
| `-B, --base` | `BASE` | | Base URL prefix for link rewriting |
| `-d, --data` | `DATA` | `./` | Directory with Markdown files |
| `-c, --cache` | `CACHE` | `false` | Enable in-memory page cache |
| `-t, --title` | `TITLE` | `false` | Show title from frontmatter or filename |

## URL Routing

- `/` and `/path/` serve `index.md` from the corresponding directory
- `/page` resolves to `page.md`
- No `.md` extension needed in URLs

## Frontmatter

Pages can include YAML frontmatter:

```markdown
---
title: My Page
tags: [docs, example]
---

# Content here
```
